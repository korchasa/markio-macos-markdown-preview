# SDS

## 1. Intro
- **Purpose:** Define the architecture and implementation approach for Markio — a native macOS Markdown previewer with GFM + Mermaid, rendered in a confined `WKWebView`, with an on-screen line-width control.
- **Rel to SRS:** Implements [REF:fr:open | FR-OPEN], [REF:fr:multidoc | FR-MULTIDOC], [REF:fr:gfm | FR-GFM], [REF:fr:mermaid | FR-MERMAID], [REF:fr:highlight | FR-HIGHLIGHT], [REF:fr:line-width | FR-LINE-WIDTH], [REF:fr:live-reload | FR-LIVE-RELOAD], [REF:fr:appearance | FR-APPEARANCE], [REF:fr:offline | FR-OFFLINE], [REF:fr:find | FR-FIND], [REF:fr:math | FR-MATH], [REF:fr:frontmatter | FR-FRONTMATTER], [REF:fr:inline-html | FR-INLINE-HTML], [REF:fr:toc | FR-TOC], [REF:fr:code-copy | FR-CODE-COPY], [REF:fr:session-restore | FR-SESSION-RESTORE], [REF:fr:local-links | FR-LOCAL-LINKS], [REF:fr:quicklook | FR-QUICKLOOK], [REF:fr:ai-artifacts | FR-AI-ARTIFACTS], [REF:fr:compare | FR-COMPARE].

## 3. Components

### 3.1 App shell [ANC:sds:app-shell]
- **Purpose:** `DocumentGroup(viewing: MarkdownDocument.self)` — one native window per file, with system File ▸ Open / Open Recent and state restoration. Window tabbing is disabled (`NSWindow.allowsAutomaticWindowTabbing = false` in `AppDelegate`) → strictly one window per document. Each window hosts its own render surface + a bottom-bar reading control. Implements [REF:fr:open | FR-OPEN], [REF:fr:multidoc | FR-MULTIDOC], [REF:fr:appearance | FR-APPEARANCE], [REF:fr:session-restore | FR-SESSION-RESTORE].
- **Session restore decision:** relaunch must reopen the documents open at quit deterministically. `AppDelegate` opts into secure state restoration (`applicationSupportsSecureRestorableState → true`) and sets `NSQuitAlwaysKeepsWindows = true` in the app's defaults domain (a `set`, not `register(defaults:)` — the registration domain loses to the user's NSGlobalDomain "Close windows when quitting" setting; the app domain overrides it). Document reopen + sandbox access are then fully system-managed (`DocumentGroup`/AppKit restoration) — no app-side session list, no bookmark code, no extra entitlement. Chosen over an own security-scoped-bookmark session store: platform-native, far less code, no new MAS review surface; trade-off: behavior rides on AppKit restoration machinery (verified in a real `.app`).
- **Interfaces:** `ContentView(document:fileURL:)` per window owns a `DocumentModel`. Opens originate from the system Open panel / Finder / Dock / `open` (handled by `DocumentGroup`), drag-drop onto a window (`@Environment(\.openDocument)` → new window), a command-line argument in dev (`AppDelegate` → `NSDocumentController.shared.openDocument`), or a clicked relative Markdown link ([REF:sds:local-links | LocalLinkNavigator] → the same `NSDocumentController` path, so one-window-per-document and Open Recent behave identically). No welcome screen: a fresh launch shows the system Open panel. Windows open at 900×820 on first launch; the system restores frames thereafter.
- **Deps:** AppKit, SwiftUI, UniformTypeIdentifiers.

### 3.1a MarkdownDocument [ANC:sds:markdown-document]
- **Purpose:** Read-only `FileDocument` carrying the file's Markdown text; the unit each `DocumentGroup` window is built around. Implements [REF:fr:multidoc | FR-MULTIDOC], [REF:fr:open | FR-OPEN].
- **Interfaces:** `readableContentTypes = [md, markdown, plainText]`; `writableContentTypes = []` (never writable → no Save, never dirty); `init(data:)` decodes UTF-8 and throws on invalid bytes (fail fast); `fileWrapper(configuration:)` throws (read-only).
- **Deps:** SwiftUI, UniformTypeIdentifiers.

### 3.1c WindowTitleSetter [ANC:sds:window-title]
- **Purpose:** Show the document's full path in the title bar instead of the bare file name `DocumentGroup` defaults to. Implements [REF:fr:multidoc | FR-MULTIDOC].
- **Interfaces:** `NSViewRepresentable` (zero-size) backed by `TitlePinningView`: on `viewDidMoveToWindow` it KVO-observes `NSWindow.title` and re-asserts the full path on every change (the document machinery re-syncs the file name asynchronously, so a one-shot set loses). Clears `representedURL` because, while set, AppKit shows the file name regardless of `title` — this deliberately drops the proxy icon.
- **Deps:** AppKit, SwiftUI.

### 3.1d ReadOnlyMenuCommands + MenuArtifactCleaner [ANC:sds:menu-commands]
- **Purpose:** Trim the auto-generated `DocumentGroup` menu to the read-only viewer surface. Implements [REF:fr:menu | FR-MENU].
- **Interfaces:** Two cooperating pieces. (1) `ReadOnlyMenuCommands: Commands`, attached via `.commands { }`, does the *semantic* removal contractually with the two groups that hold document-write commands: `.newItem`→∅ (New) and `.saveItem`→∅ (Save/Save As/Duplicate/Rename/Move To/Revert/Share + Close/Close All; Open Recent survives). Nothing else is touched — the Edit menu is left standard, so macOS disables inapplicable items on non-editable content and localizes everything for free (no custom buttons). (2) `MenuArtifactCleaner` interposes an `NSMenuDelegate` on each top submenu that forwards to SwiftUI's delegate, then removes *structural* artifacts the emptied groups leave behind — a title-less placeholder item (`title==""`, no view/submenu/action; AppKit draws it "NSMenuItem") and orphaned leading/trailing/duplicate separators (skipping hidden items). Matches no selector, identifier, or title → locale- and version-robust.
- **Decision:** "Good enough", not surgical. Per-item control over the SwiftUI-owned `DocumentGroup` menu proved fragile; the stable path is to use the two contractual groups and leave the rest standard. An earlier attempt replaced `.pasteboard` with custom Copy/Select All buttons — dropped, because it hard-coded English titles and forfeited the free localization. Localization itself is declarative: `CFBundleLocalizations` (en, ru) in `Info.plist`; AppKit then renders standard items in the system language. Residual: Edit's input items (AutoFill, Start Dictation, Emoji & Symbols) remain. NB: command effects appear only in a real `.app` bundle (`make app`/`prod`); `make dev` builds a degraded menu.
- **Deps:** SwiftUI, AppKit.

### 3.1b DocumentModel [ANC:sds:document-model]
- **Purpose:** Per-window state: owns the window's `PreviewController`, `FileWatcher`, and reading width; renders the document text and re-renders on appearance/live-reload. One instance per `DocumentGroup` window (no shared singleton). Implements [REF:fr:live-reload | FR-LIVE-RELOAD], [REF:fr:appearance | FR-APPEARANCE], [REF:fr:line-width | FR-LINE-WIDTH].
- **Interfaces:** `start(text:url:)` (one-time page setup + render + arm watcher), `setWidth(px)`, `appearanceChanged(dark:)`. Live reload re-reads `url` off-main and re-renders.
- **Deps:** AppKit, SwiftUI.

### 3.2 FileLoader [ANC:sds:file-loader]
- **Purpose:** Read file contents off the main thread; hand raw Markdown text to the render host. Implements [REF:fr:open | FR-OPEN].
- **Interfaces:** `load(url) -> String`; emits updates on change events from FileWatcher.
- **Deps:** Foundation.

### 3.3 FileWatcher [ANC:sds:file-watcher]
- **Purpose:** Watch the open file for external modification; trigger reload. Implements [REF:fr:live-reload | FR-LIVE-RELOAD].
- **Interfaces:** `watch(url, onChange)`; debounced; handles atomic-save replace (re-arm on vnode delete/rename).
- **Deps:** Dispatch / FSEvents.

### 3.4 WebViewHost [ANC:sds:webview-host]
- **Purpose:** Own the `WKWebView` (`PreviewController`, surfaced to SwiftUI via `PreviewView`); load the shell as a **self-contained document** (`ResourceLocator.selfContainedHTML()` inlines every `vendor/` `<link>`/`<script src>`, then `loadHTMLString(_:baseURL: nil)` — no `file:` subresource reads); push Markdown source and width into the page. Implements [REF:fr:gfm | FR-GFM], [REF:fr:mermaid | FR-MERMAID], [REF:fr:highlight | FR-HIGHLIGHT], [REF:fr:offline | FR-OFFLINE].
- **Interfaces:** `loadTemplate() async throws` (throws if the shell navigation fails, so a failed load is distinguishable from success), `render(_:)`, `setContentWidth(_ chars: Int) -> String?`, `setDark(_:)`, `outline() -> [TOCItem]`, `scrollToHeading(_:)`, `currentSection() -> String?`, `setScrollY(_ y: Double) -> Double?` (returns the browser-clamped applied position — [REF:fr:session-restore | FR-SESSION-RESTORE]) — all via `callAsyncJavaScript` into the page world; bridge failures are logged (`os.Logger`), not swallowed. Five one-way `WKScriptMessageHandler`s (each registered through a weak `ScriptMessageProxy` because the user-content controller retains its handlers strongly): `markioTOC` — the page pushes the current heading id on scroll → `onCurrentSectionChange` ([REF:fr:toc | FR-TOC]); `markioCopy` — a copy-button click pushes the block's raw code text, validated and written to an injected `NSPasteboard` (init parameter, default `.general`; tests pass a uniquely named pasteboard) with a `onCodeCopied` callback as test/UI hook ([REF:fr:code-copy | FR-CODE-COPY]); `markioScroll` — the page pushes the debounced scroll offset (validated: non-negative finite number) → `onScrollPositionChange` ([REF:fr:session-restore | FR-SESSION-RESTORE]); `markioLink` — a click on a relative Markdown link pushes the raw href (untrusted page data; validated non-empty string) → `onLinkActivated`, resolved and opened natively ([REF:fr:local-links | FR-LOCAL-LINKS], [REF:sds:local-links | LocalLinkNavigator]); `markioSyncScroll` — the page pushes the live scroll fraction while compare sync is on (validated: finite 0…1) → `onSyncScroll` ([REF:fr:compare | FR-COMPARE], [REF:sds:compare | CompareCoordinator]); everything else is driven native→web, and web links are intercepted by the navigation delegate (`decidePolicyFor` → `LinkPolicy`: in-page `file:` / external via `NSWorkspace` / block). Network disabled via `WKWebView` config + navigation policy.
- **Decision:** Load via `loadHTMLString(baseURL: nil)` over `loadFileURL(allowingReadAccessTo:)`. Under the Mac App Store App Sandbox the confined WebContent process never completes a `file:` shell navigation → the preview stays blank (reproduced on a signed TestFlight build). Reading assets app-side (own-bundle reads are always allowed) and handing WebKit one inlined document avoids `file:` entirely. Separately, the `com.apple.security.network.client` entitlement is required even though nothing is fetched: without it the sandboxed WebContent helper fails to launch at all. Rendering stays fully offline — the navigation delegate blocks every web load.
- **Deps:** WebKit, os.

### 3.5 LineWidthControl [ANC:sds:line-width]
- **Purpose:** Bottom-bar slider bound to the reading width in **characters**; persists the value. Implements [REF:fr:line-width | FR-LINE-WIDTH].
- **Interfaces:** `ContentView.bottomBar` — a `.safeAreaInset(edge: .bottom)` bar holding a stepped `Slider` (range `minWidth…maxWidth`, `step`) plus a `N ch` readout. `ContentWidthStore` reads/writes `UserDefaults` key `contentWidthChars` (Int, characters) and `clamp`s to `[40, 200]` snapped to the 20-step grid; default 80. `DocumentModel.setWidth` → `PreviewController.setContentWidth(chars)` → JS `setContentWidth` sets `--content-width: <n>ch`.
- **Decision:** Character width via the CSS `ch` unit (absolute, window-independent up to the physical window-width cap — a column wider than the window is clipped to it) over a px or window-fraction width; a stepped slider (presets) over free-form. New `UserDefaults` key (`contentWidthChars`) so legacy px values are not misread. Note: `ch` = advance of the '0' glyph (a wide glyph), so a column set to N fits ≳N typical letters per line.
- **Deps:** SwiftUI, Foundation.

### 3.7 FindBar [ANC:sds:find-bar]
- **Purpose:** Native find-in-document surface. Implements [REF:fr:find | FR-FIND].
- **Interfaces:** Three cooperating pieces. (1) `DocumentModel` holds the find state (`findPresented`, `findQuery`, `findResult`) and the operations (`openFind`/`closeFind`/`runSearch`/`findNext`/`findPrev`), each delegating to `PreviewController`; on live-reload / appearance re-render it re-applies the active query. (2) `ContentView.findBar` — a floating HUD via `.overlay(alignment: .top)` (not a bar that pushes content): a `Capsule` filled with `windowBackgroundColor` (hairline border + shadow) holding a magnifier glyph, a borderless `FindTextField` (an `NSTextField` with no bezel/focus ring), an "N of M" counter (`FindResult.counterText`, unit-testable formatter), a separator, and round `HUDButton` prev/next/close controls. The field auto-focuses on open; Enter → next, Shift+Enter → previous, Esc/✕ → close. (3) `FindCommands: Commands` + a `FocusedValue` (`documentModel`) route a single app-level Find / Find Next / Find Previous menu (⌘F / ⌘G / ⌘⇧G) to the focused window's model.
- **Bridge:** `PreviewController.search(_:) / findNext() / findPrev() / clearSearch()` call the page's `search`/`findNext`/`findPrev`/`clearSearch` via `callAsyncJavaScript`, returning `FindResult(count, current)`. Highlighting is done in the page — the "web view owns content rendering" rule; the shell (bar, menu, keys) stays native.
- **Matching (page):** two-phase. Phase 1 concatenates the visible text nodes (`collectTextNodes` exclusions: svg, KaTeX, code-copy UI, script/style) into one haystack with an offset→node map, inserting NUL (U+0000) between nodes whose nearest block ancestor differs (`BLOCK_TAGS` walk) — a query cannot contain NUL, so a match never spans a block boundary. Phase 2 finds case-insensitive occurrences and wraps them from the LAST to the FIRST (`splitText` only trims the tail of a mapped node, keeping earlier offsets valid). `__find.matches` holds logical matches — arrays of 1+ `<mark class="markio-find">` segments; a match crossing highlight.js token spans or inline formatting splits into several segments, all toggled to `markio-find-current` together; `setCurrent` scrolls the first segment into view. `clearSearch` unwraps every mark and `normalize()`s parents.
- **Minimap (page):** `div#markio-find-minimap` on `<body>` (outside `#content`, so `render()`'s innerHTML swap never orphans it) — a fixed 10 px strip at the right edge, `pointer-events: none`, one absolutely-positioned tick per logical match at `top = (firstSegment pageY / document scrollHeight) · 100%`; the current tick carries `markio-find-tick-current`. Rebuilt by every `search()`, repositioned on window `resize`, removed by `clearSearch()`. Indication-only (no click-to-jump) per the minimalism rule.
- **Deps:** SwiftUI, AppKit, WebKit.

### 3.8 TOCSidebar [ANC:sds:toc-sidebar]
- **Purpose:** Toggleable native sidebar with the document's heading tree: click-to-jump, current section highlighted while scrolling. Implements [REF:fr:toc | FR-TOC].
- **Interfaces:** Split along the find-feature boundary — native chrome, page-side content mechanics. (1) Page (`template.html`): after every render a DOM walk assigns GitHub-style slug ids (deduped `-1`, `-2`, …) to `#content` `h1–h6` and caches them in `__toc`; `getOutline()` → `[{level, text, id}]`; `scrollToHeading(id)` → `scrollIntoView({block:'start'})`; `getCurrentSection()` → id of the last heading at/above the viewport-top threshold; a synchronous `scroll` listener (no rAF — it doesn't fire in offscreen web views) posts the id to `webkit.messageHandlers.markioTOC` when it changes (guarded: absent handler → no-op). (2) `PreviewController`: `TOCItem` (level/text/id), `outline()`/`scrollToHeading(_:)`/`currentSection()`, `onCurrentSectionChange` callback fed by the `markioTOC` handler. (3) `DocumentModel`: `tocVisible` (persisted via `TOCStore`, `UserDefaults` key `tocSidebarVisible`, default hidden — a global reading preference like the line width), `outline`, `currentHeadingID`, `toggleTOC()`, `jumpToHeading(_:)`; `refreshOutline()` runs after every render (start / live reload / appearance), next to `reapplyFindIfActive`, so the tree survives re-renders ([REF:fr:live-reload | FR-LIVE-RELOAD]). (4) `TOCSidebar` view: fixed-width (220 pt) `ScrollView`+`LazyVStack` of plain buttons, indent 12 pt × (level−1), current row accent-highlighted and kept visible via `ScrollViewReader`; empty outline → dimmed placeholder. (5) `TOCCommands`: `View ▸ Show/Hide Table of Contents` (⌥⌘S) through the shared `FocusedValue(\.documentModel)`.
- **Decision:** Native sidebar in an `HStack` (not `NavigationSplitView`, which would drag a toolbar into the deliberately toolbar-less window; recorded as a possible future evolution) and a single read-only page→native message handler for the scroll-spy (not native polling — laggy and wasteful). This supersedes the earlier "no `WKScriptMessageHandler`" bridge decision; the surface stays minimal and auditable: one-way handlers with single-string payloads (`markioCopy` later joined it on the same pattern — [REF:fr:code-copy | FR-CODE-COPY]).
- **Deps:** SwiftUI, WebKit, Foundation.

### 3.9 ScrollPositionStore [ANC:sds:scroll-store]
- **Purpose:** Persist each document's last scroll position across launches so a reopened document (relaunch, Open Recent, ⌘O) restores the reading place. Implements [REF:fr:session-restore | FR-SESSION-RESTORE].
- **Interfaces:** `position(for: URL) -> Double?`, `setPosition(_: Double, for: URL)`. `UserDefaults` key `scrollPositions`: `[path: {y, seq}]`, bounded at 200 entries with least-recently-written eviction (`seq` is a monotonic sequence number — deterministic under same-second writes). Malformed stored values read as absent (the viewer must open regardless). Injectable suite for tests.
- **Flow:** page `scroll` listener (true 250 ms debounce) → `markioScroll` string → `PreviewController.onScrollPositionChange` → store write (continuous saving survives force quit); `DocumentModel.start` reads the store after the window's first render and calls `setScrollY` once (`started`-guarded — never re-fires while the user reads).
- **Deps:** Foundation.

### 3.10 LocalLinkNavigator [ANC:sds:local-links]
- **Purpose:** Make links inside rendered documents functional: relative Markdown links open in a new window, `#anchor` links scroll in place, `other.md#section` does both. Implements [REF:fr:local-links | FR-LOCAL-LINKS].
- **Interfaces:** Split along the established boundary — page intercepts, native decides. (1) Page (`template.html`): a delegated click listener on `#content` reads the RAW `href` attribute; `#fragment` → `preventDefault` + `scrollToHeading(decodeURIComponent(fragment))` (TOC slug ids — [REF:fr:toc | FR-TOC]); a scheme'd (`http:`, `mailto:`, `ftp:`, …) or protocol-relative href is left untouched for the navigation delegate (external → default browser, rest blocked — `LinkPolicy` unchanged); any other relative href is prevented, and only `.md`/`.markdown` targets post the raw href to the one-way `markioLink` handler — default-deny, non-Markdown relative clicks stay dead. (2) `LocalLink.swift`: `LocalLink(fileURL:anchor:)` + pure `LocalLinkResolver.resolve(href:documentURL:)` — accepts scheme-less relative paths with Markdown extensions only (percent-decoded, `..` traversal allowed, absolute paths rejected), splits the optional fragment. (3) `LocalLinkNavigator` (`@MainActor`, app-wide `shared`): `follow(href:from:)` resolves, records the anchor as pending (in-memory `[path: anchor]`, consume-once, nothing persisted), and opens the file; `attach(_:)` registers each started `DocumentModel` weakly as a `LocalLinkTarget` (`documentURL` + `navigate(toAnchor:)`). A freshly opened window consumes its pending anchor after the first render (`DocumentModel.start`, anchor wins over the saved scroll position); an already-open window is scrolled directly through its registered target (it never re-renders). The open primitive is injected for tests.
- **Sandbox decision (honest):** The MAS build has only `files.user-selected.read-only` — by App Sandbox semantics a sibling file that was never user-selected is not readable, so a silent "just open it" is impossible without new entitlements. Chosen model: per-file powerbox grant. `follow` checks `FileManager.isReadableFile`; readable targets (previously opened, recents — access is system-managed, plus everything in unsandboxed dev runs) open directly via `NSDocumentController.shared.openDocument` (the same path as command-line opens); a denied target gets an `NSOpenPanel` pre-pointed at it (`directoryURL`, Markdown content types, explanatory message) — the user's one "Open" click IS the grant; cancel is a no-op. A missing file is indistinguishable from a denied one inside the sandbox, so it shows the same panel (the user sees the absence and cancels). Chosen over folder-level security-scoped bookmarks (one grant per repo): bookmarks need a new entitlement (`bookmarks.app-scope`), persistent state with staleness handling, and app-store-factory signing coordination — recorded as the evolution path in the task's Follow-ups, not v1. Signed-build behavior is verified manually (`documents/checklists/local-links.md`); unit tests cover the grammar, interception, and anchor hand-off.
- **Deps:** AppKit, UniformTypeIdentifiers, Foundation.

### 3.12 CompareCoordinator [ANC:sds:compare]
- **Purpose:** Side-by-side compare: pair two document windows, tile them left/right, and mirror scrolling live at the same fraction of each document's own scrollable height. Parallel reading only — no diff semantics. Implements [REF:fr:compare | FR-COMPARE].
- **Interfaces:** Split along the established boundary — page reports, native decides. (1) Page (`template.html`): `setCompareSync(enabled)` toggles live reporting; while enabled, a synchronous scroll listener posts `String(fraction)` (`scrollY / max(1, scrollHeight − innerHeight)`, clamped 0…1) to the one-way `markioSyncScroll` handler on EVERY scroll event (no debounce — the debounced `markioScroll` persistence channel is separate and untouched); `setScrollFraction(f)` applies a mirrored position and arms a ONE-SHOT suppression flag so the resulting scroll event is consumed, not echoed (the flag is only armed when the position actually moves >1 px — a no-move apply must not swallow the next genuine user scroll); `getScrollFraction()` seeds a fresh link. (2) `PreviewController`: fifth `ScriptMessageProxy` (`markioSyncScroll`, validated finite 0…1 string → `onSyncScroll`), `setCompareSync(_:)`, `setScrollFraction(_:) -> Double?`, `scrollFraction() -> Double?` — best-effort, logged like the rest of the bridge. (3) `CompareCoordinator` (`@MainActor`, app-wide `shared`, open primitive injected for tests): weak target registry (`attach(_:)` from `DocumentModel.start` AFTER the first render, so a pending pair always links against a scrollable page); `beginCompare(from:)` → powerbox `NSOpenPanel` ("Compare" click = sandbox grant, as in [REF:sds:local-links | LocalLinkNavigator]) → `NSDocumentController` open (already-open file focuses — [REF:fr:multidoc | FR-MULTIDOC] intact; picking the initiator's own file is a no-op) → link directly or park ONE pending pair completed by `attach`; `link` enables sync on both sides, seeds the peer from the initiator's fraction, tiles via pure `tileFrames(in:)` (left/right halves of the initiator's screen `visibleFrame`, odd point to the right); `scrollChanged(from:fraction:)` applies to the peer under a re-entrancy guard; `unlink(for:)` disables sync on both. Pairs are weak symmetric peers — deallocation (window close) drops the pair and disables sync on the survivor; session-only, nothing persisted. (4) `DocumentModel: CompareTarget` (`hostWindow`, `setCompareSyncEnabled` → `@Published isCompared`, `applyScrollFraction`, `currentScrollFraction`) + `startCompare()`/`stopCompare()`. (5) `CompareCommands: Commands` — `Window ▸ Compare Side by Side…` / `Stop Comparing` via the shared `FocusedValue(\.documentModel)`, `CommandGroup(before: .windowArrangement)`.
- **Decision:** Two normal document windows paired by a coordinator (not a two-pane compare window): keeps window-per-document and ALL per-window state (TOC, find, width, scroll persistence) intact for free, and stays minimal — the only new machinery is one message channel and one coordinator. Proportional fraction mapping per the product decision (v1); heading-based structural alignment recorded as a follow-up. Mirroring deliberately follows ANY scroll, including TOC jumps and find navigation ("paired reading follows any navigation"). Echo suppression lives in the page (one-shot flag) because the echo is asynchronous — a native flag alone cannot see it; the native re-entrancy guard covers only the synchronous path.
- **Deps:** AppKit, UniformTypeIdentifiers, WebKit (via `PreviewController`).

### 3.6 Vendored web bundle [ANC:sds:vendor]
- **Purpose:** Offline rendering assets under `Sources/MarkioEngine/Resources/vendor` + `Resources/template.html`, owned by the shared `MarkioEngine` library target (SwiftPM resource bundle `Markio_MarkioEngine.bundle`; also home of `ResourceLocator` and `MarkdownFileReader`) so the app and the Quick Look extension use one engine. Implements [REF:fr:gfm | FR-GFM], [REF:fr:mermaid | FR-MERMAID], [REF:fr:highlight | FR-HIGHLIGHT], [REF:fr:offline | FR-OFFLINE], [REF:fr:math | FR-MATH], [REF:fr:frontmatter | FR-FRONTMATTER], [REF:fr:inline-html | FR-INLINE-HTML], [REF:fr:ai-artifacts | FR-AI-ARTIFACTS].
- **Interfaces:** `template.html` exposes JS entrypoints `render(markdown)` (preserves the window scroll position across the re-render — captured on entry, restored after Mermaid settles, so live reload / appearance re-renders never yank the reader; [REF:fr:live-reload | FR-LIVE-RELOAD]), `setContentWidth(chars)` (sets `--content-width` in `ch`), `getContentWidth()`, `setDark(bool)`, the find API `search(query)`/`findNext()`/`findPrev()`/`clearSearch()` (each returns `{count, current}`), the scroll API `setScrollY(y)` (returns the browser-clamped applied value) / `getScrollY()` ([REF:fr:session-restore | FR-SESSION-RESTORE]), and the compare-sync API `setCompareSync(enabled)` / `setScrollFraction(f)` / `getScrollFraction()` ([REF:fr:compare | FR-COMPARE], [REF:sds:compare | CompareCoordinator]); reads CSS var `--content-width`. Native calls them via `callAsyncJavaScript`. Copied flat to the bundle root (template + `vendor/` siblings); `ResourceLocator.selfContainedHTML()` reads them app-side and inlines every `<link>`/`<script src>` into a single document loaded with `baseURL: nil` (no `file:` subresource reads under the App Sandbox — see [REF:sds:webview-host]).
- **Deps (pinned, committed):** markdown-it 14.1.0 + markdown-it-task-lists 2.1.1 (wrapped as a browser global), highlight.js 11.10.0 (common langs) with github light/dark themes, mermaid 11.6.0 (UMD, `securityLevel:strict`), github-markdown-css 5.8.1, KaTeX 0.16.11 (`katex.min.js` global + `katex.css` with fonts inlined as `data:` URIs → self-contained, resolves with no base URL), DOMPurify 3.2.4 (`purify.min.js` global).
- **Inline-HTML rule:** markdown-it runs with `html:true`; the full render output passes `sanitizeHtml()` — a DOMPurify allowlist gate (`USE_PROFILES: {html, mathMl}` + `FORBID_TAGS: ['style']`) — **before** `innerHTML` assignment, so `<script>`, event-handler attributes, and `javascript:` URLs are stripped and `<img onerror>` never fires (insert-then-clean would fire it). Generated markup (KaTeX HTML+MathML, hljs spans, `pre.mermaid`, task-list `<input>`) passes the gate; a missing DOMPurify global throws → render rejects loudly, nothing unsanitized is inserted. Decision: vendored DOMPurify over a hand-rolled DOMParser walker (mXSS class, MathML/SVG namespace pitfalls) and over CSP hardening (overkill for the offline shell). The `<script src="vendor/dompurify/purify.min.js">` tag must keep the exact `src`-first, attribute-less form `ResourceLocator.inlineVendor()`'s regex matches. Implements [REF:fr:inline-html | FR-INLINE-HTML].
- **Math rule:** an inline `mathPlugin` (in `template.html`) registers markdown-it rules `math_inline` (`$…$`, after `escape`) and `math_block` (`$$…$$`, after `blockquote`) so math tokenizes at parse time (emphasis/escape never touch formula content; `$` in code spans stays literal; standard `isValidDelim` no-digit-after-close money guard). Both rules render via `katex.renderToString(tex, { throwOnError:false, trust:false, displayMode })`. Implements [REF:fr:math | FR-MATH].
- **Copy-button rule:** the markdown-it `highlight` callback stamps tagged fences with `data-lang` (attribute-escaped; survives DOMPurify's default `ALLOW_DATA_ATTR`); after every render `decorateCodeBlocks()` wraps each `pre.hljs:not(.markio-frontmatter)` in `div.markio-codeblock` (positioned wrapper — the `<pre>` itself has `overflow:auto`) and appends hover-revealed `div.markio-code-ui` (badge from `data-lang` + a Copy `<button>`). One delegated click listener on `#content` (survives re-renders) posts the sibling `<code>`'s `textContent` — markup-free raw code — to `markioCopy` (guarded no-op headless) and flashes "Copied" ~1.5 s. Excluded: `pre.mermaid`, the frontmatter box. Find's `collectTextNodes` skips `.markio-code-ui`, so UI text is never a match. Decision: clipboard write is native (`NSPasteboard` via the handler), not in-page `navigator.clipboard` — WKWebView does not expose the Clipboard-API permission plumbing under the App Sandbox, the same platform lesson as the shell-load decision in [REF:sds:webview-host]. Implements [REF:fr:code-copy | FR-CODE-COPY].
- **Frontmatter rule:** a `frontMatterPlugin` (in `template.html`) registers a block rule `front_matter` `before('table')` — ahead of `hr`/`lheading` — that matches only at document start (`startLine===0`, zero indent): opening/closing lines entirely `-` (≥3). The captured YAML renders through the vendored highlight.js `yaml` grammar into `<pre class="hljs markio-frontmatter"><code>` (a captioned, bordered metadata box; CSS in `template.html`). No closing fence → falls through to default parsing (best-effort). Reuses highlight.js — no new dependency. Implements [REF:fr:frontmatter | FR-FRONTMATTER].
- **ANSI rule:** detection is content-based in the markdown-it `highlight` callback — any non-mermaid fence containing ESC (U+001B) + `[` renders via the hand-rolled `renderAnsi()` (no vendored library) into `<pre class="hljs markio-ansi">`, **replacing** syntax highlight (agent logs are pasted untagged or mis-tagged; a language-tagged fence with ESC bytes deliberately trades language highlighting for color rendering). **Supported SGR subset (honest):** reset `0`; bold `1`, dim `2`, italic `3`, underline `4`, strikethrough `9`; off `22/23/24/29`; fg `30–37`/`90–97`/default `39`; bg `40–47`/`100–107`/default `49`; 256-color `38;5;n`/`48;5;n`; truecolor `38;2;r;g;b`/`48;2;r;g;b`. Everything else — inverse `7`, blink, cursor movement/erase CSI, OSC (titles, hyperlinks), Fe escapes — is stripped, never shown raw; the parser is total (a malformed sequence consumes its own bytes only — NFR Reliability). Named colors 0–15 → palette classes `ansi-fg-N`/`ansi-bg-N` scoped under `.markio-ansi` (VS Code terminal palettes, dark override via `prefers-color-scheme` like the hljs themes); 256-cube/grayscale + truecolor → inline `style` (survives the DOMPurify gate, which allows `style` attributes by default — test-asserted). Copy on ANSI blocks yields the visible escape-free text (`code.textContent`) — a deliberate deviation from byte-exact fence content, decided at variant selection; find's two-phase search crosses ANSI spans like hljs token spans. Implements [REF:fr:ai-artifacts | FR-AI-ARTIFACTS].
- **Diff rule:** CSS-only. The vendored hljs common bundle already carries the `diff` grammar (aliases `patch`): `hljs-addition`/`hljs-deletion` spans cover each `+`/`-` line and the github themes color them per scheme. `template.html` stretches those spans to full-width blocks (`display:inline-block; min-width:100%; box-sizing:border-box`) so the theme background becomes a line background; `min-width` (not `width`) keeps it covering horizontally scrolled long lines while the `<pre>` keeps `overflow:auto`. Hunk headers stay `hljs-meta`. Implements [REF:fr:ai-artifacts | FR-AI-ARTIFACTS].
- **Wrap rule:** long unbroken tokens: prose and inline code already wrap via github-markdown-css (`word-wrap:break-word` on `.markdown-body`); code blocks deliberately keep scrolling (`overflow:auto`). The one real gap was table cells — `break-word` does not shrink a cell's intrinsic width, so a long path forced the whole table into a horizontal scroller; `template.html` adds `overflow-wrap:anywhere` on `td`/`th` so the token wraps inside the column. Implements [REF:fr:ai-artifacts | FR-AI-ARTIFACTS].

### 3.11 Quick Look preview extension [ANC:sds:quicklook]
- **Purpose:** Space on a `.md` file in Finder renders it with Markio's engine (GFM, Mermaid, KaTeX) before the app is ever opened. Implements [REF:fr:quicklook | FR-QUICKLOOK].
- **Interfaces:** SwiftPM `executableTarget MarkioQuickLook` (depends on `MarkioEngine`). `PreviewViewController` (`@objc(PreviewViewController)`, pinned — `NSExtensionPrincipalClass` references it verbatim) — `NSViewController` + `QLPreviewingController`; `preparePreviewOfFile(at:completionHandler:)` reads via `MarkdownFileReader` (strict UTF-8), then `QuickLookRenderHost` — a minimal `WKWebView` owner: `loadTemplate()` (self-contained shell via `ResourceLocator`), `render(_:)` (throws → error reaches the completion handler → system fallback preview), `setDark(_:)` (best-effort from `effectiveAppearance`). No message handlers; the navigation delegate allows only the initial `loadHTMLString` navigation and cancels everything after — a QL preview never navigates and never opens links.
- **Decision — view-based, not data-based:** the data-based QL API (`QLPreviewReply` HTML) executes no JavaScript, which Mermaid requires; a view-based controller hosting its own `WKWebView` runs the full engine (JS in QL-hosted WKWebViews is proven by shipping extensions: csv-quick-look, quicklookjs, MermaidViewer). Known QL/WKWebView platform limits and how they are dodged: network entitlement ignored → irrelevant (offline engine, assets inlined); external links open in-preview → all post-load navigations cancelled; fullscreen `@media` dark-mode bug → theme driven by explicit `setDark`, not media queries.
- **Packaging (no Xcode):** the binary links with entry `_NSExtensionMain` (Foundation) via `linkerSettings` unsafe flags — how Xcode links app-extension products; a stub `main.swift` satisfies SwiftPM's executable-entry requirement and never runs. `make app` assembles `Contents/PlugIns/MarkioQuickLook.appex` (binary + own copy of `Markio_MarkioEngine.bundle` — cross-bundle reads under the extension sandbox are not guaranteed — + `packaging/MarkioQuickLook-Info.plist`) and **ad-hoc signs the appex only** with `packaging/MarkioQuickLook.entitlements` (app-sandbox; extensions must be signed+sandboxed for pluginkit to load them). The extension Info.plist declares `QLSupportedContentTypes = [net.daringfireball.markdown]` only (never `public.plain-text`), imports the markdown UTI, and carries `CFBundleSupportedPlatforms`/`DT*` keys — ExtensionFoundation crashes on hand-assembled bundles without them. Distribution signing is app-store-factory's job (nested appex signed first — recorded in the task's Follow-ups).
- **Verification:** structure + signature + input-gate acceptance are automated (see SRS FR-QUICKLOOK); the rendered panel is walked manually (`documents/checklists/quicklook.md`). Local registration may need `pluginkit -a <appex>` + `pluginkit -e use -i dev.markio.app.quicklook`; note `qlmanage -o` (headless) cannot host view-based extensions — use GUI `qlmanage -p` or Finder.
- **Deps:** Quartz (QuickLookUI), WebKit, MarkioEngine.

## 5. Logic
- **Algos:** Render = read file → `render(markdown)` in page → md parser (`html:true`) produces HTML (a leading `---`…`---` YAML frontmatter tokenized first and rendered as a highlighted metadata box; math tokens `$…$`/`$$…$$` rendered inline via `katex.renderToString` at parse time) → **DOMPurify allowlist sanitize** ([REF:fr:inline-html | FR-INLINE-HTML]) → `innerHTML` → `mermaid.run()` over `.language-mermaid` blocks → highlight over remaining code blocks (an ANSI-bearing fence renders SGR colors via `renderAnsi` instead of syntax highlight — [REF:fr:ai-artifacts | FR-AI-ARTIFACTS]) → restore window scroll captured at render entry (WebKit clamps the scroll while Mermaid SVGs are momentarily their short source text; the restore keeps the reader's place on live reload — [REF:fr:live-reload | FR-LIVE-RELOAD]). Width = bottom-bar control → `setContentWidth(chars)` sets `--content-width: <n>ch`; content column `max-width: var(--content-width)` → an absolute N-character reading measure, window-independent. TOC = post-render heading walk (slug ids) → native pulls `getOutline()` after every render; page pushes the current section id on scroll via `markioTOC`; clicks jump via `scrollToHeading(id)` ([REF:fr:toc | FR-TOC]). Copy = post-render `decorateCodeBlocks()` wrap pass → delegated click posts raw `<code>` text via `markioCopy` → `PreviewController` writes `NSPasteboard` ([REF:fr:code-copy | FR-CODE-COPY]).
- **Rules:** Confine `WKWebView` to bundled file URLs; intercept external links → open in default browser via `NSWorkspace` (never navigate the view). Local links: the page intercepts clicks — `#anchor` scrolls in place, relative `.md` hrefs go to the native resolver/opener (new window; powerbox grant when sandbox-denied), everything else stays blocked ([REF:fr:local-links | FR-LOCAL-LINKS]). Compare: while a pair is live, each side posts its scroll FRACTION per scroll event over `markioSyncScroll`; the coordinator applies it to the peer via `setScrollFraction`, whose one-shot page-side suppression consumes the resulting event — no echo, no loop ([REF:fr:compare | FR-COMPARE]). Debounce file-change events. Load file I/O off the main thread; render calls marshaled to main.

## 6. Non-Functional
- **Scale/Fault/Sec/Logs:** Off-main-thread file reads keep UI responsive on large docs. Malformed Markdown → best-effort render, no crash. Security: no network (offline FR), minimal JS bridge — native→web calls, link interception, and five one-way page→native message handlers with single-string payloads (`markioTOC`, [REF:fr:toc | FR-TOC]; `markioCopy`, [REF:fr:code-copy | FR-CODE-COPY]; `markioScroll`, [REF:fr:session-restore | FR-SESSION-RESTORE]; `markioLink` — untrusted href gated by a default-deny resolver, [REF:fr:local-links | FR-LOCAL-LINKS]; `markioSyncScroll` — live compare scroll fraction, validated 0…1, [REF:fr:compare | FR-COMPARE]). Logging via `os.Logger`, subsystem `dev.markio`.

## 7. Constraints
- **Packaging:** `make app`/`make prod` assemble a real `Markio.app` (binary + SwiftPM resource bundle `Markio_MarkioEngine.bundle` + `packaging/Info.plist`, built under `.build/`), including the embedded Quick Look extension (`Contents/PlugIns/MarkioQuickLook.appex`, ad-hoc-signed — [REF:sds:quicklook]). The bundle (bundle id + `CFBundleDocumentTypes` for `.md`/`.markdown`) is what gives macOS single-instance behavior, "Open With" routing, and one-window-per-document. The app icon is compiled at this step: `iconutil -c icns packaging/AppIcon.iconset` → `Contents/Resources/AppIcon.icns`, referenced by `CFBundleIconFile` ([REF:fr:icon | FR-ICON]). `make dev` runs the raw SwiftPM binary (no bundle → a separate process per launch, no icon) for fast iteration.
- **Swift 6 concurrency (AppKit glue):** (1) Make AppKit-bridging coordinators `@MainActor` — a global-actor-isolated class is implicitly `Sendable`, clearing "non-Sendable capture / sending self" errors. (2) `NotificationCenter`/KVO closures are `@Sendable`; when registered with `queue: .main`, run the isolated work via `MainActor.assumeIsolated { … }`. (3) A nonisolated `deinit` cannot touch non-Sendable stored properties — invalidate observers in `viewDidMoveToWindow(nil)`/on rebind, not `deinit`.
- **Platform:** macOS 14+ (Swift 6 language mode). Min raised from 13 → 14 to use modern SwiftUI `onChange` and keep a zero-warning build.
- **Simplified:** Read-only documents (no Save/edit); minimal chrome — one bottom-bar control (reading width in characters), no toolbar. System appearance only (no custom theme picker in v1). No welcome screen — fresh launch shows the system Open panel. TOC sidebar width is fixed (220 pt, not resizable) in v1.
- **Deferred:** Print/export, custom themes — explicitly out of v1 scope per minimalism priority. (Window-per-document, Open Recent, and state restoration come free from `DocumentGroup`; window tabbing is deliberately disabled — one document = one window.)
